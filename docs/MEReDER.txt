# **3. Observações e Melhorias**

1. **Clareza na responsabilidade:**

   * Cada RF/RNF está mapeado a entidades específicas; isso ajuda no desenvolvimento modular e testes unitários.

2. **Extensibilidade:**

   * Adicionar novos upgrades ou inimigos exige apenas criar uma nova instância da entidade correspondente e configurar spawn/padrões.

3. **Mínimo para MVP:**

   * Player + Bullet + Enemy + Wave + Upgrade + GameState → núcleo suficiente.
   * RF e RNF de média prioridade (feedback visual, efeitos extras) podem ser adicionados depois.

4. **Facilidade de testes:**

   * É possível simular GameState e waves em C++ sem render, facilitando debug e ajustes de balanceamento.
